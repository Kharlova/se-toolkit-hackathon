Context

End User: Plant owners (beginners and experienced) who want to maintain healthy plants through consistent watering schedules.

Problem: Plant owners often forget when they last watered their plants or struggle to maintain consistent watering schedules, leading to under/over-watering and plant health issues.

Solution: A simple web app to track plants, set watering schedules, and mark watering completion with visual status indicators — all in one click.

Implementation

Stack: Python (http.server) backend, SQLite database, HTML/CSS/JS frontend, Docker for deployment.

  • Version 1: Core feature — add plants, set watering frequency, view schedules, mark as watered, one-click watering.
  • Version 2: Added SQLite persistence, visual status indicators (overdue/due today/good), responsive UI, Docker deployment on university VM.
  • TA Feedback Addressed: Database persistence for data durability, improved UX with color-coded status indicators, production-ready Docker deployment.
